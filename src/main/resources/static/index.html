📕 一个非常非常简易的使用文档

本项目专注于自动化任务执行，帮助用户通过可视化的方式定义复杂的网页交互操作，零代码实现自动化。

一、 XPath 的获取

  • XPath是 Web 元素在页面中的唯一路径，是自动化操作的基础。
  • 如何获取：
    1. 使用 F12 快捷键进入浏览器开发者工具。
    2. 点击开发者工具左上角的 “选择元素” 按钮（通常是一个带箭头的方块图标）。
    3. 在页面上点击需要定位的元素，开发者工具将自动定位到该元素的 HTML 标签。
    4. 右键点击该标签，选择“Copy” -> “Copy full XPath” 即可获得完整的 XPath。

二、 流程编排

  • 最佳实践： 在执行任务前，最好点击 “校验步骤” ，确保所有必填字段均已填写，避免运行错误。
  • 等待时间： 建议在 第一步或任何可能导致页面加载的步骤之后，设置一个适当的 “等待时间(ms)” (例如 1000ms 或更久)，以防止因页面响应过慢而找不到目标标签。

三、 循环操作与占位符

循环操作（如遍历点击、动态循环）是实现复杂任务的关键。它们依赖于占位符来动态改变 XPath

  • 遍历操作（循环点击 / 循环输入 / 循环获取内容）

    • 默认占位符： 支持使用 `{i}` 和 `{index}`。
    • 默认起始值： 默认为 1 。
    • 应用场景： 主要用于操作页面上按顺序排列的元素，XPath中可以写作：//div[@class="list"]/a[{index}]。

  • 动态循环

    • 默认占位符： 支持使用 `{i}` 和 `{index}`。
    • 默认起始值： 默认为 0 。
    • 增量： 为这个占位符每次循环后增加多少 。
    • 自定义占位符： 动态循环允许您自定义一个占位符名称（例如：`item_id`），该占位符的值将可以被其所有子任务共用。
    • 子任务支持： 动态循环的子任务可以完全使用父级定义的占位符进行 XPath 定位或值输入。

  • 占位符表达式支持

    占位符 `{i}` 和 `{index}` 均支持数学表达式运算，极大提高了任务编写的灵活性。

    格式示例:

    {index+1}: 索引加 1
    {index-1}: 索引减 1
    {index*3}: 索引乘以 3
    {index*5+10}: 复杂的代数运算

    您可以根据 XPath 的数字规律，通过表达式来定位非连续或有规律间隔的元素。
